Microsoft Azure Static Website Deployment Step by Step

26th June 2023

Step-by-Step Deployment Procedure for a Static Website on Azure:

Project Setup:

- Create the project for your static website.
- Build your website with all the necessary files, such as home.html, CSS files, and JavaScript files.

Repository Creation:

- Go to GitHub and create a new repository for your website.
- Choose the option to upload files, as it is an easier way to add your website files.
- Upload all the required files (.html, .css, .js) to the repository.
- Once the files are uploaded, your repository is ready for deployment.

Azure Account Creation:

- Visit the Microsoft Azure website, preferably through the Azure for Students program.
- Click on the provided link to create an account with Azure.
- Sign up using your personal email address, not your school's email address (e.g., kipruto@gmail.com).

Account Activation:

- After creating the Azure account, follow the instructions to activate your student benefits, which include $100 in credit.
- Scroll down on the Azure portal and look for the option to connect with your school's email account.
- Click on the link and provide the necessary details.
- You will be directed to the Microsoft 365 landing page, where you need to create an account.
- Once the account is created, return to the Azure portal to access your $100 credit.

Website Deployment:

- Sign in to the Azure portal.
- On the landing page, click on "Create a resource."
- Select "Web App" and click on "Create."
- Fill in the required details for the web app and choose the "Static Web App" option.
- Click on "Create" to proceed.
- Provide the necessary details again, and select the GitHub radio button as the deployment source.
- Connect your GitHub account to Azure.
- Select your organization (usually your name) and choose the repository you created earlier.
- Set the branch to "Main."
- Click on "Review + Create" and confirm the details.
- Azure may take a few minutes to deploy the website using a CDN (Content Delivery Network).
- Once deployed, copy the URL of your website and share it wherever needed, such as on Moodle.
