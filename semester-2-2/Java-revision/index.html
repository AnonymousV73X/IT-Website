Object-oriented Programming with JAVA

  • Introduction
    Object-oriented programming (OOP) is a programming paradigm that organizes and conceptualizes a program as a set of interacting objects

    Objects and Classes:
    Objects are instances of classes, representing real-world entities or concepts. They encapsulate data and behavior.

    Classes, on the other hand, are blueprints or templates that define the structure and behavior of objects.

    Encapsulation:
    Encapsulation is the concept of bundling data (variables) and methods (functions) within a class, hiding the internal implementation details from the outside world.
    It allows for better control and protection of data, ensuring that it can only be accessed through defined methods.

    Methods and Variables:
    Methods are blocks of code (functions) defined within a class to perform specific tasks or actions. They represent the behavior of objects.

    Variables, on the other hand, are used to store data and represent the state of an object.

    Inheritance:
    Inheritance is a mechanism in Java where one class (subclass) inherits properties and behaviors from another class (superclass). The subclass can access the public and protected members of the superclass, allowing code reuse and establishing an "is-a" relationship between classes.

    A constructor is a special method in a class that is automatically called when an object of the class is created.

    Message Passing and Polymorphism:
    Message Passing is the process of objects communicating with each other by invoking methods.
    When a method is called on an object, it sends a message to that object requesting the execution of the method. This allows objects to interact and collaborate in a Java program.

    Polymorphism is the ability of a method to take on multiple forms based on the type of object it is called on.
    - True Polymorphism: Different classes have methods with the same name but different behaviors (e.g., draw() for Circle and Square classes).
    - Parametric Polymorphism: Within a class, the same method name is used with different signatures (different parameters).
    - Overloading: Operators (+,-,/,*, etc.) can be applied to multiple types, such as int, floats, strings, etc.
    - Overriding: Subclasses replace the behavior of a parent class with new or modified behavior.
  • Advantages of Java
    Object-Oriented Paradigm:
    Java's object-oriented nature allows for easy extension and promotes code reusability.

    Platform Independence:
    Java's platform independence enables distribution of byte code over the web, making it executable on any platform with the help of the JVM.

    Simplicity and Learnability:
    Java's simplicity makes it easy to learn and master, especially for those familiar with OOP concepts.

    Strong Security:
    Java's secure features facilitate the development of virus-free and tamper-free systems with strong authentication techniques based on public-key encryption.

    Architectural Neutrality and Portability:
    Java's architectural-neutrality and portability ensure compatibility across various processors and systems, allowing smooth execution in different environments.

    Robustness and Error Prevention:
    Java's robustness minimizes error-prone situations through rigorous compile-time and runtime checks.

    Multithreading Capability:
    Java's multithreading capability allows developers to create applications that efficiently perform multiple tasks simultaneously.

    Rapid Development with Interpretation:
    Java's interpreted nature speeds up development and linking processes, making it a rapid and analytical development environment.

    High Performance with JIT Compilation:
    Java's use of Just-In-Time compilers ensures high performance in executing code.

    Distributed Environment:
    Java's design for the distributed environment of the internet makes it well-suited for building networked applications.

    Dynamic Adaptability:
    Java's dynamic features enable adaptability to an evolving environment, carrying extensive runtime information for verifying and resolving accesses to objects.
  • Implementing Java
    Basic JAVA syntax
    Java has a specific set of rules and conventions for writing code, such as using semicolons to end statements, defining classes with the "class" keyword, and using curly braces to enclose blocks of code.

    public class HelloWorld {
      public static void main(String[] args) {
        System.out.println("Hello, World!");
      }
    }

    First JAVA program
    When starting to learn Java, the first program usually prints a simple message like "Hello, World!" on the screen. It serves as an introduction to the basic structure of a Java program.

    public class FirstProgram {
      public static void main(String[] args) {
        System.out.println("This is my first Java program!");
      }
    }

    Modifiers
    Modifiers in Java are keywords that provide additional information about classes, methods, or variables. They can specify access levels (e.g., public, private) or behavior (e.g., static, final) for different elements in the code.

    public class ExampleClass {
      // Private access modifier restricts access to only within this class
      private int privateVariable;
      // Public access modifier allows access from anywhere
      public void publicMethod() {
        // Code for the method
      }
      // Static modifier allows the method to be accessed without creating an instance of the class
      public static void staticMethod() {
        // Code for the static method
      }
    }

    Addition of two numbers program
    A common beginner's program in Java is one that takes two numbers as input from the user or pre-defined in the code and then adds them together, displaying the result on the screen. It's a simple way to understand input/output and basic arithmetic operations in Java.

    import java.util.Scanner;
    public class AddTwoNumbers {
      public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Enter the first number: ");
        int num1 = scanner.nextInt();

        System.out.print("Enter the second number: ");
        int num2 = scanner.nextInt();

        int sum = num1 + num2;

        System.out.println("The sum of " + num1 + " and " + num2 + " is: " + sum);
      }
    }
  • Classes, Objects & Methods
    Java class:
    A Java class is a blueprint that defines the structure and behavior of objects.

       class ClassName {
         // Fields: Used to store data
         // Constructor: Special method to initialise a method
         // Methods: Used to perform operations
       }

    Java Objects:
    Objects are instances of classes representing real-world entities or concepts.

    Creating an Object in Java:
    To create an object, use the following syntax:
       ClassName object = new ClassName(); Example using a class named "Bicycle":
       Bicycle sportsBicycle = new Bicycle();
       Bicycle touringBicycle = new Bicycle();
    In the example above, we create two Bicycle objects: "sportsBicycle" and "touringBicycle". These objects are instances of the Bicycle class and can have their own data and perform operations defined in the class.

    Creating Method:
    Considering the following example to explain the syntax of a method:
       public static int methodName(int a, int b) {
         // body
       }
    Here,
       public static: modifier.
       int: return type
       methodName: name of the method
       a, b: formal parameters
       int a, int b: list of parameters

    Method Overloading:
    Method overloading is when multiple methods in the same class have the same name but different parameter lists. It allows a single method name to represent different behaviors based on the number or type of parameters passed to it.

       public int add(int a, int b) {
         return a + b;
       }
       public double add(double a, double b) {
         return a + b;
       }

    Method Overriding:
    Method overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass. The method in the subclass has the same name, return type, and parameters as the one in the superclass, effectively replacing its behavior.

       class Shape {
         public void draw() {
           System.out.println("Drawing a shape");
         }
       }
       class Circle extends Shape {
         @Override
         public void draw() {
           System.out.println("Drawing a circle");
         }
       }
    In this example, the Circle class overrides the draw method from its superclass Shape to provide a specific implementation for drawing a circle.
  • Inheritance
    Inheritance:
    A fundamental concept in OOP where a new class (subclass) inherits properties and behaviors from an existing class (superclass)

    // Superclass
    class Animal {
      String name;

      Animal(String name) {
        this.name = name;
      }

      void eat() {
        System.out.println(name + " is eating");
      }
    }

    // Subclass
    class Dog extends Animal {
      Dog(String name) {
        super(name); // Call the superclass constructor
      }

      void bark() {
        System.out.println(name + " is barking");
      }
    }

    // Main class
    public class Main {
      public static void main(String[] args) {
        // Creating an instance of Dog class
        Dog dog = new Dog("Buddy");

        // Calling methods from the Dog class
        dog.eat(); // This method is inherited from the Animal class
        dog.bark(); // This method is specific to the Dog class
      }
    }

    Output:
    Buddy is eating
    Buddy is barking

    In this example, we have two classes, Animal and Dog.

    The super keyword:
    The super keyword in Java is used to access and call members of the superclass from the subclass. It enables method overriding and allows the subclass to reuse functionality provided by the superclass.

    Example:
    class Animal {
      void makeSound() {
        System.out.println("Roar");
      }
    }

    class Dog extends Animal {
      void makeSound() {
        super.makeSound(); // Calls the makeSound() method of the superclass
        System.out.println("Bark");
      }
    }

    public class Main {
      public static void main(String[] args) {
        Animal animal = new Animal();
        animal.makeSound(); // Output: Roar

        Dog dog = new Dog();
        dog.makeSound(); // Output: Roar Bark
      }
    }

    "Is-A" relationship:
    It represents inheritance, where one class (subclass) extends another class (superclass) and inherits its properties and behaviors. It establishes a hierarchical relationship between classes.

    "Has-A" relationship:
    It represents composition or aggregation, where one class contains an object of another class as a member. It allows a class to use the features of another class without inheriting from it.
  • Encapsulation & Polymorphism
    Encapsulation:
    Bundling data (balance) and methods (deposit, withdraw) within the BankAccount class, hiding internal implementation details.

    class BankAccount {
      private double balance; // Private variable to store the account balance

      public void deposit(double amount) {
        // Deposit logic
        balance += amount;
      }

      public void withdraw(double amount) {
        // Withdrawal logic
        if (amount <= balance) {
          balance -= amount;
        } else {
          System.out.println("Insufficient balance.");
        }
      }
    }

    Polymorphism:
    The draw() method takes different forms based on the object it is called on, showing different shapes being drawn.

    class Shape {
      void draw() {
        System.out.println("Drawing a shape");
      }
    }

    class Circle extends Shape {
      void draw() {
        System.out.println("Drawing a circle");
      }
    }

    class Square extends Shape {
      void draw() {
        System.out.println("Drawing a square");
      }
    }

    Explanation:
    Encapsulation: The BankAccount class encapsulates the data (balance) and methods (deposit, withdraw) to hide the implementation details and provide controlled access to them.

    Polymorphism: The Shape class and its subclasses (Circle and Square) demonstrate polymorphism, where the draw() method takes different forms depending on the type of object it is called on, allowing different shapes to be drawn with the same method name.

    Advantages of encapsulation:
    - Data Protection: Encapsulation hides internal data and allows controlled access, preventing unauthorized modifications.
    - Code Flexibility: Internal implementation details can be changed without affecting the external interface, promoting code flexibility.
    - Code Maintenance: Encapsulation makes it easier to maintain and update code since changes are localized within the class.
    - Reusability: Encapsulation enables code reuse through well-defined interfaces, increasing code reusability.
    - Security: Encapsulation enhances security by controlling access to sensitive data and methods, protecting the integrity of the code.
    - Design Modularity: Encapsulation promotes a modular design, making the code more organized and easier to understand.
    - Team Collaboration: Encapsulation allows different team members to work independently on different parts of the code without interfering with each other's work.

  • Abstraction
    Abstraction:
    Abstraction is a process of hiding the implementation details and showing only functionality to the user.

    abstract class Shape {
      abstract void draw();
    }

    class Circle extends Shape {
      void draw() {
        System.out.println("Drawing a circle");
      }
    }

    class Square extends Shape {
      void draw() {
        System.out.println("Drawing a square");
      }
    }

    In this example, we have:
    - An abstract class Shape with an abstract method draw(). The abstract method does not have any implementation in the abstract class. Instead, it is implemented in its subclasses Circle and Square, which extend the Shape class. These subclasses provide specific implementations for the draw() method, showing how each shape is drawn.

    - The abstract class Shape hides the complexity of drawing shapes and focuses on the essential feature of the shapes' ability to be drawn. This is an example of abstraction in Java, where we only show what the shapes can do (draw), without exposing how they do it (the specific drawing logic).
