Screen 1A-i · Login
OTP-Only Authentication
No password ever. One tap sends a free SMS OTP — mirroring M-Pesa, eCitizen, and KCB flows that parents already use daily.
  • Kenyan phone number with +254 flag pre-selected
  • 6-digit OTP with live countdown timer
  • USSD fallback *384# for feature phone users
  • Swahili toggle — primary, not secondary
Jakob's Law: Zero new mental model required. Same flow as every Kenyan bank and government service.
Screen 1A-ii · National ID
One-Time Identity Verification
Step 2 of 2 — first login only. Parent enters their National ID number and full name. An optional ID card photo speeds up admin approval. This step never repeats on subsequent logins.
  • Progress bar + step dots show exactly where user is
  • Navy header maintains context from Step 1
  • Amber info card explains why — reduces friction
  • ID photo upload is optional — admin can verify with number alone
  • Encryption reassurance note reduces privacy anxiety
  • "Skip photo" path ensures no user is blocked
Addresses As-Is Scenario 1: Faith's identity was never verified at any point in the original process. This screen ensures every parent is a named, verifiable person before any authorization can be issued — UC-01 precondition.
Screen 1B · Home
Parent Dashboard
Single-glance status: active student, pickup circle status, one large CTA. The most important action (authorize) is always one tap away.
  • Welcome hero with real-time stats
  • Student card with school + form
  • Amber CTA for authorize — highest contrast
  • Activity feed with timestamped events
  • Upcoming event reminder with urgency signal
Screen 1C · Pickup Circle
Pre-Verified Trusted People
Parents build their circle once, valid for the academic year. Each person verified via National ID — removes the need for handwritten letters entirely.
  • Clear approved / pending status for each member
  • Relationship + masked phone number shown
  • One-tap add new member
  • Offline fallback note for elderly relatives
  • Admin must approve each member before they're active
Addresses As-Is Scenario 1: Faith's identity was never verified at any point. This screen ensures every picker is a named, verifiable person before any authorization is issued.
Screen 1D · Authorize
Time-Bound Authorization in ~90 seconds
Parent selects event, picker, and time window. System generates QR + OTP automatically. Replaces handwritten letters and WhatsApp messages to teachers.
  • Event type dropdown (Closing Day / Emergency / Visiting Day)
  • Picker selected from pre-approved circle only
  • Time window limits token validity precisely
  • Optional instructions for the picker
  • Dual delivery: QR to app + OTP via SMS
Covers UC-02: The entire authorization process that previously took 40+ minutes of phone calls is reduced to one form, under 90 seconds.

Screen 2A · Picker Token
Gate Access — QR + OTP Dual Mode
What Faith sees when she arrives at the school gate. She shows the QR code to the teacher's scanner — or reads the 6-digit OTP aloud if connectivity is an issue.
  • QR code is the primary — fast, no reading required
  • Live countdown timer creates urgency awareness
  • OTP backup for low-data or elderly pickers
  • Token is single-use — consumed after one successful scan
  • Works fully offline — pre-downloaded at auth time
Covers UC-03 + Mzee Peter scenario: An elderly grandfather with a feature phone receives the OTP via plain SMS and reads it aloud at the gate. No app, no data, no confusion.

Screen 3A · Gate Scan
Fast Gate Verification — Under 10 Seconds
Teacher processes 20–30 pickups in a 3-hour window. The scanner fills the screen, class list is instantly visible, and there's no login overhead per student.
  • Animated scan line — clear visual feedback
  • Manual OTP entry as fallback (no-data scenario)
  • Live progress counter: 14 of 30 released
  • Class list shows pending vs done at a glance
  • Offline QR verification — no live internet needed
Designed for time pressure: Closing day gate queue observed at 18–25 min per family. SafePass targets <90 seconds per pickup, including confirmation tap.
Screen 3B · Authorized
Clear, Unambiguous AUTHORIZED State
Large ✅, green background, picker photo, student name, and time validity — designed for low-literacy gate security who need a visual result, not a text-first result.
  • Picker photo shown for face match by watchman
  • One large green CTA: "Confirm Release"
  • Token automatically consumed after confirmation tap
  • Parent push notification sent instantly
  • Event written to D4 Audit Logs
Screen 3C · Declined
Clear Rejection + Escalation Path
Failure is just as important as success. Teacher knows exactly why and what to do next — contact admin or request emergency override. Student stays safe.
  • Reason for decline clearly stated (expired / invalid / reused)
  • Student is not released — implicit safe default
  • Direct shortcut to contact admin
  • Emergency override request button
  • All declined attempts logged with timestamp + reason
